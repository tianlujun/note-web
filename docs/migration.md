# Obsidian → HTML Migration Pipeline

## Overview

```
Obsidian vault (TOS: tos:tianlujun-default/Obsidian/)
  → migrate-vault.py (local: /root/migrate-vault.py)
  → HTML vault (local: /root/notes/)
  → TOS (tos:tianlujun-default/notes/)
  → NERD (/root/notes/)
  → note-web rebuild (API call)
```

## Prerequisites

- Python 3: `markdown-it-py`, `PyYAML`, `python-dateutil`
- `rclone` with TOS remote configured
- Migration script: `/root/migrate-vault.py`

## Step-by-Step

### 1. Run migration script (locally)

```bash
cd /root
python migrate-vault.py
```

Reads: `tos:tianlujun-default/Obsidian/` (TOS vault)
Writes: `/root/notes/` (local HTML vault)

The script:
- Converts all `.md` files to static `.html`
- Resolves `[[wikilinks]]` to relative `.html` links
- Copies attachments (images, etc.) to `attachments/` dirs
- Generates `_index.html` (directory index with breadcrumb nav)
- Generates `link-index.json` (node/edge graph for the note-web knowledge graph)

### 2. Sync to TOS

```bash
rclone sync /root/notes/ tos:tianlujun-default/notes/ --delete-after
```

This pushes the HTML vault to TOS for backup and serving.

### 3. Sync to NERD

```bash
rclone sync /root/notes/ root@172.245.147.11:/root/notes/
```

### 4. Rebuild note-web cache

```bash
curl -X POST http://127.0.0.1:8082/api/cache/rebuild \
  -H "Authorization: Bearer $TOKEN"
```

The `link-index.json` is generated by the migration script; the API call tells note-web to re-index the HTML files into its SQLite cache.

## Migration Script Details

**Location:** `/root/migrate-vault.py` (local to Hermes)

**Key transformations:**
- Frontmatter `title`, `created`, `updated`, `type`, `tags` → HTML metadata block
- Wikilinks `[[note]]` → `<a href="note.html">` (same-dir) or `../dir/note.html` (subdir)
- Case normalization: all dir/file names lowercased in output
- Breadcrumb: auto-generated from directory hierarchy

**Output structure:**
```
notes/
  00-scheme/          (Schema docs)
  01-inbox/           (Inbox)
  02-projects/        (Projects)
  03-practice/        (Practice)
  04-learning/        (Learning — with attachments/)
  05-entities/        (Entities)
  _index.html          (root index)
  link-index.json      (knowledge graph)
```

## Wikilink Resolution Rules

1. `[[note]]` → looks for `*/note.html` (any subdir) or `*/note/index.html`
2. `[[subdir/note]]` → looks for `subdir/note.html` (same level) or `subdir/note/index.html`
3. `[[shame_pink/]]` → links to `shame_pink/_index.html` (directory index)
4. Attachments: `![[image.png]]` → `<img src="attachments/image.png">`

## Common Issues

### Wikilink unresolved
Check that target note filename matches (lowercased). The script logs `UNRESOLVED:` for wikilinks that couldn't be resolved.

### Attachments not found
Attachments must be copied to `attachments/` directories in the same subdir as the note. The migration script handles this if `attachments/` dir exists in the Obsidian vault.

### Rebuild 401 Unauthorized
`ACCESS_TOKEN` env var is empty by default. Set a strong token in the systemd service file or environment, then use the same token in the Bearer header.
